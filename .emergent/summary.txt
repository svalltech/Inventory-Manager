The user requested an online/offline inventory management application for sports clothing, featuring detailed SKUs (color, fabric, size, design, price). It needs to be an independent, lightweight, scalable module, interactive with future billing and e-commerce apps. Key requirements included:
- **Offline/Online Sync:** PWA with Service Workers, IndexedDB, and background sync.
- **Inter-Module Communication:** RESTful APIs ().
- **Inventory Filters & Search:** Advanced multi-select filters, price range, text search.
- **Authentication & Access:** Role-based (Admin/Staff/Viewer).
- **Inventory Details:** Gender (Male/Female), Size (XS(36)-XL(44)), MRP, Selling Price, Cost Price (optional), Weight (grams). Both MRP and Selling Price must be displayed.
- **Brand & Warehouse Management:** Input for Brand and Warehouse, brand-specific warehouse selection, allowing new entries. Brand/Warehouse fields to be prioritized at the top of forms.
- **Data Export:** Download inventory data as Excel, PDF, Word, with selectable fields and template management.
- **Data Import:** Bulk upload/update inventory data via Excel.
- **Dashboard:** Primary dashboard post-login, table display with Category, Product Name, SKU, Design, Size, Quantity, Total Value (Selling price). Sort options, pagination (20, 50, 100, ALL), and table totals.
- **Table Interaction:** Search option for any variable directly in table columns, and an edit option (pencil icon) next to each row for direct data updates.
</product_requirements>
